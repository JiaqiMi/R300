# VisionSnapshotLayer 多障碍独立TTL修复

## 修复的问题

旧逻辑如果在收到新的非空帧时执行：

```cpp
active_points_.clear();
```

会产生：

- A消失后进入空场景：没有触发clear，A可保持到TTL；
- A消失但B立即出现：B属于非空帧，先clear全部，A立即消失。

本版本改为：

1. 每帧先把几何点聚成障碍簇；
2. 与已有障碍轨迹按质心距离匹配；
3. 匹配到的轨迹只更新自身轮廓并刷新TTL；
4. 未匹配的新障碍新增轨迹；
5. 未匹配的旧障碍继续保持到自己的5秒TTL；
6. 空帧不立即清空任何轨迹。

## 验证

启动后日志应包含：

```text
hold=5.00s cluster=0.45m association=0.80m
```

测试顺序：

```text
A出现 → A退出并立即出现B
```

预期：

```text
A继续保留到其最后一次观测后的约5秒
B立即加入并单独保持
A与B互不清除
```
