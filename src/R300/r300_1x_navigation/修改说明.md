# R300 视觉障碍快照层修改说明

## 解决的问题

旧链路使用：

`obstacle_scan -> 360° costmap_scan -> costmap_2d::ObstacleLayer`

其中存在两个状态保持阶段：

1. Python 节点按 `base_link` 的激光束编号保持障碍；
2. ObstacleLayer 再使用射线清除和障碍标记维护历史栅格。

车辆原地转向时，第一阶段保存的旧激光束会被按当前 `base_link` 姿态重新解释；同时，360°伪清除扫描并不是真实相机自由空间观测。这会造成旧黑团残留、新红线不落图等现象。

新链路为：

`detections -> obstacle_scan -> VisionSnapshotLayer -> inflation_layer -> DWA`

VisionSnapshotLayer 将当前障碍点在消息时间戳处转换到 `odom`，在 `odom` 中保持 1 秒，并在每个 costmap 周期重建整张视觉层。过期点必然删除，新点在下一周期标记，不依赖伪造的 360°射线清除。

## 关键参数

- `vision_snapshot_layer/hold_time_s: 1.0`：最终安全保持时间，仍为 1 秒；
- `subject1_vision_obstacles.yaml/hold_time_s: 0.20`：只用于跨越相邻检测帧，不再承担最终保持；
- `/r300_vision/active_obstacle_scan`：插件实际持有的障碍调试扫描，网页应显示该话题；
- `always_send_full_costmap: true`：此时 `/costmap_updates` 为 0 Hz 正常，应查看完整 costmap 频率。

## 安装

将本补丁目录中的文件按原目录覆盖到：

`~/r300_ws/src/R300/r300_1x_navigation/`

然后：

```bash
cd ~/r300_ws
source /opt/ros/noetic/setup.bash
catkin_make --pkg r300_1x_navigation -j4
source devel/setup.bash
```

检查插件注册：

```bash
rospack plugins --attrib=plugin costmap_2d | grep r300_1x_navigation
```

启动台架测试：

```bash
cd ~/r300_ws/src/R300/r300_1x_navigation/scripts/one_key
./start_r300_vision_nav.sh --no-base --no-rviz
```

启动网页：

```bash
python3 ~/r300_ws/src/R300/r300_1x_navigation/scripts/costmap_web_viewer.py _port:=8088
```

诊断：

```bash
python3 ~/r300_ws/src/R300/r300_1x_navigation/scripts/diagnose_vision_costmap.py _duration_s:=20
```

## 预期现象

- 当前障碍出现：`active_obstacle_scan` 有红线，完整 costmap 下一周期出现黑色核心；
- 车辆转向：红线和黑团在 `odom` 中对应同一物体，不随 `base_link` 错位漂移；
- 目标消失：红线和黑团继续保持最多 1 秒，随后同步消失；
- 视觉话题断流：插件保留最后障碍并将 costmap 标为非 current，使 move_base 失败保护。
