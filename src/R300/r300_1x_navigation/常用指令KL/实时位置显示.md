
启动轨迹显示节点

第一步：
在 1X 已经正常启动后，新开终端：

source /opt/ros/noetic/setup.bash
source /home/amov/amovcar/devel/setup.bash

python3 /home/amov/amovcar/src/R300/r300_1x_navigation/scripts/ins_trajectory_viewer.py

第二步
打开 RViz

再新开一个终端：

source /opt/ros/noetic/setup.bash
rviz

RViz 里按下面设置：

Fixed Frame：odom

    然后依次添加：

    Add → Path
    Topic：/one_x/trajectory

    Add → Odometry
    Topic：/one_x/odom

    Add → Marker
    Topic：/one_x/trajectory_info

    建议调一下：

    Path 的 Line Style：Billboards
    Path 的 Line Width：0.05

    你会看到：

    轨迹线：INS 实时位置轨迹
    箭头：车辆当前姿态
    文字：实时经纬度、航向、东向E、北向N坐标


第三步
每次测试前清空轨迹
rosservice call /one_x/trajectory_reset

建议每次测试前都清一次，例如：

静止漂移测试前清一次
直行测试前清一次
转圈测试前清一次
自主导航测试前清一次



先把 RViz 调整成容易看的样子

左侧展开 Odometry，把：

Covariance

前面的勾去掉。

然后在右侧 Views 里，把视角改成：

Type: TopDownOrtho

这样就是从正上方看轨迹。

再点击顶部：

Focus Camera

然后点一下黄色车体箭头或绿色轨迹附近，让画面自动放大到车周围。

建议把轨迹线宽调成：

Path -> Line Width = 0.08 或 0.1
这个图应该怎么理解

按照你当前 1X 驱动代码：

x = East，东向
y = North，北向

所以正确情况下：

车头朝北向前走：轨迹应主要沿 y 增加方向变化
车头朝东向前走：轨迹应主要沿 x 增加方向变化

但是注意：当前画面是斜视角，而且没有明显坐标轴，所以不要直接凭“屏幕上往上还是往下”判断东南西北。建议再加一个：

Add -> TF

然后打开 odom、base_link 坐标轴显示。

ROS 常见坐标颜色是：

红色 X：东向
绿色 Y：北向
蓝色 Z：向上

这样你就能直接看出轨迹往哪个方向漂。
