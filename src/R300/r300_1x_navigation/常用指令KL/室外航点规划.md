# 0. 当前系统状态

当前使用的导航链路：

```text
定位：1X 惯导
1X 串口：/dev/ttyACM0

雷达：MID360
雷达话题：/scan
雷达坐标系：livox_frame

底盘：Scout / CAN can0

导航：move_base
航点执行：waypoint_executor
速度安全守卫：cmd_vel_guard
```

完整数据链路：
```text
1X
→ /one_x/odom
→ odom -> base_link

MID360
→ /scan
→ move_base local_costmap

map -> odom
→ odom -> base_link
→ base_link -> livox_frame

waypoint_executor
→ move_base
→ /subject1/cmd_vel_raw
→ cmd_vel_guard
→ /subject1/cmd_vel_safe
→ scout_base
```

重点理解：
```text
/subject1/cmd_vel_raw   = move_base 原始规划速度
/subject1/cmd_vel_safe  = 安全守卫过滤后给底盘的速度
```

不动车测试时：
```text
cmd_vel_raw 可以有速度
cmd_vel_safe 必须为 0
```

# 9. 不动车航点测试
开始航点：
```bash
rosservice call /subject1/start_waypoints
```
查看 move_base 原始速度：
```bash
rostopic echo /subject1/cmd_vel_raw
```
查看最终安全速度：
```bash
rostopic echo /subject1/cmd_vel_safe
```
正确状态：
```text
cmd_vel_raw 可以有非零速度
cmd_vel_safe 必须为 0
```
原因：
```text
motion_enabled=false
```
如果小车附近有障碍，可能出现：
```text
potential collision
valid plan could not be found
Rotate recovery can't rotate in place
```
这通常说明：
```text
MID360 障碍物数据已经进入 local_costmap
move_base 避障正在生效
不是代码坏了
```

# 13. 常见问题

## 13.1 1X 没有数据

检查串口：
```bash
ls -l /dev/ttyUSB* /dev/ttyACM* 2>/dev/null
```
当前应使用：
```text
/dev/ttyACM0
```
检查诊断：
```bash
rostopic echo -n 1 /one_x/diagnostics
```
## 13.2 move_base 报 map 和 base_link 不连通


检查：
```bash
rosrun tf tf_echo odom base_link
rosrun tf tf_echo map base_link
```
如果缺 `map -> odom`，检查：
```bash
grep -n "subject1_map_to_odom\|map odom" /home/amov/amovcar/src/R300/r300_1x_navigation/launch/subject1_waypoint_nav.launch
```

## 13.3 有障碍时规划失败

现象：
```text
potential collision
valid plan could not be found
Rotate recovery can't rotate in place
```
处理：
```bash
rosservice call /subject1/cancel_waypoints
rosservice call /move_base/clear_costmaps "{}"
rosservice call /subject1/start_waypoints
```
如果仍然失败，先清空小车周围障碍物。

## 13.4 raw 有速度，但 safe 为 0

这是正常保护状态。
常见原因：
```text
motion_enabled=false
/scan 超时
/one_x/odom 超时
move_base 命令超时
```
当不动车测试时，safe 为 0 是正确的。
